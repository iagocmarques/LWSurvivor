# Plan: Adapt Agent Architecture Blueprint for Kilo Code

## Objective
Adapt the `agent-architecture-template.md` (Claude Code blueprint) into a complete Kilo Code equivalent, preserving the agent structure and orchestration topology while mapping all Claude Code concepts to Kilo Code's configuration model.

## Key Mappings (Claude Code → Kilo Code)

| Concept | Claude Code | Kilo Code |
|---------|-------------|-----------|
| Root instruction file | `CLAUDE.md` | `AGENTS.md` |
| Agent definitions | `.claude/agents/*.md` | `.kilo/agent/*.md` |
| Agent frontmatter `name` | explicit field | derived from filename |
| User-invocable agents | `user-invocable: true` | `mode: primary` |
| Specialist-only agents | `user-invocable: false` | `mode: subagent` |
| Tool permissions | `.claude/settings.local.json` | `kilo.json` `permission` field |
| MCP servers | `.vscode/mcp.json` or inline | `kilo.json` `mcp` field |
| Hooks (PreToolUse, PostToolUse, Stop) | `.claude/settings.json` + `scripts/` | NO native hooks; use `permission` deny/ask + `kilo.json` instructions + commands |
| Skills | `.claude/skills/` | `.kilo/skill/*/SKILL.md` |
| Commands | n/a | `.kilo/command/*.md` |
| Model config | per-agent frontmatter | `kilo.json` `model` + per-agent `model` override |
| `Agent` tool (delegation) | Claude Code built-in | `task` tool in Kilo |
| `AskUserQuestion` | Claude Code built-in | `question` tool in Kilo |
| `TodoWrite` | Claude Code built-in | `todowrite` tool in Kilo |
| `EnterPlanMode` | Claude Code built-in | plan mode (system-reminder based) |
